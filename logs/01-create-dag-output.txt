===========================================
Exchange 2019 DAG Lab
PS1 – Create DAG (DAG01)
Server: MSEX2019 (Exchange Management Shell)
Timestamp: 2025-11-16
===========================================

Command:
    New-DatabaseAvailabilityGroup `
        -Name "DAG01" `
        -WitnessServer "WIN2019AD" `
        -WitnessDirectory "C:\DAGFileShareWitnesses\DAG01.win2019ad.ivanhwlam.com" `
        -DatabaseAvailabilityGroupIPAddresses 192.168.10.70

Output:
    WARNING: Please restart the Microsoft Exchange Information Store service 
    on each server after adding them to a DAG.

    Name                                     Servers                     WitnessServer
    ----                                     -------                     --------------
    DAG01                                    {}                          WIN2019AD.win2019ad.ivanhwlam.com


Command:
    Add-DatabaseAvailabilityGroupServer -Identity DAG01 -MailboxServer MSEX2019

Output:
    The operation completed successfully.


Command:
    Add-DatabaseAvailabilityGroupServer -Identity DAG01 -MailboxServer WIN-EX2019

Output:
    The operation completed successfully.


Command:
    Get-DatabaseAvailabilityGroup DAG01 | fl Name,Servers,WitnessServer,WitnessDirectory,DatabaseAvailabilityGroupIpAddresses

Output:
    Name                           : DAG01
    Servers                        : {WIN-EX2019, MSEX2019}
    WitnessServer                  : win2019ad.win2019ad.ivanhwlam.com
    WitnessDirectory               : C:\DAGFileShareWitnesses\DAG01.win2019ad.ivanhwlam.com
    DatabaseAvailabilityGroupIPAddresses : {255.255.255.255}

Notes:
- 255.255.255.255 表示 DAG IP 尚未綁定，屬正常現象。
- DAG 建立成功後，兩台 EX server 會自動加入 DAG 成員。
